## LAYER 1 — Executive Summary

### Real System State

The VoiceFlow Tourism PoC is a **functional multi-agent system** with a 4-layer architecture (Presentation → Application → Business → Integration) built on FastAPI + LangChain + OpenAI GPT-4. It supports both real external API integrations (Google Places, Google Routes, OpenRouteService, Overpass/OSM) and automatic fallback to local mock data.

**Key capabilities confirmed from code:**
- Speech-to-Text via Azure/Whisper with factory pattern
- NLU via OpenAI `gpt-4o-mini` with keyword fallback and shadow mode
- NER via spaCy `es_core_news_md` with provider abstraction
- 3 domain tools (Places, Directions, Accessibility) with real API providers + local fallback
- Typed pipeline via `ToolPipelineContext` (Pydantic)
- Async-native pipeline with `asyncio.gather()` for NLU+NER parallelism
- Resilience layer: circuit breaker + rate limiter + budget tracker
- Profile system: UI selector → backend resolution → prompt directives + wheelchair routing
- 156 test functions across 29 test files

### Risk Areas

1. **Security (B4):** No authentication, no HTTPS, permissive CORS — system is completely open
2. **In-memory persistence:** All conversation state is lost on restart; no concurrency protection
3. **Fixed pipeline orchestration:** Every query runs all tools (even irrelevant ones) — cost and latency concern with real API keys
4. **Profile ranking gap:** Profile affects text tone but NOT which venues are recommended — users may notice contradictions
